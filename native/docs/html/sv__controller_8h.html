SharedBuffer

The SharedBuffer merges all publishers' internal buffers into a single time-ordered interleaved schedule. Each entry points directly into the publisher's pre-built frame cache (zero copy).

This serves the same architectural role as SharedRingBuffer (teammate's implementation), but optimized for the pre-built SV use case:

  • No Boost dependency
  • No locking at runtime (schedule is immutable once built)
  • Zero-copy (pointers into publishers' frame caches)
  • Timestamp-ordered (sorted during build)
Note
When SharedRingBuffer with Boost.Interprocess is available, the SharedBuffer can be replaced without changing the rest of the architecture.
