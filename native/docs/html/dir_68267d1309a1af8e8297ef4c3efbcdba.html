Detailed Description

SV-ASN: IEC 61850-9-2LE Sampled Values Encoder

📌 Overview

This project provides ASN.1 BER encoding functions for creating IEC 61850 Sampled Values (SV) messages. It is designed for embedded systems and follows the IEC 61850-9-2 Light Edition (LE) specification.

🎯 Purpose

SV (Sampled Values) protocol is used in electrical substations for:

  • Merging Units (MU) to publish voltage/current measurements
  • IEDs (Intelligent Electronic Devices) to receive measurements for protection/control
  • High-speed, time-synchronized data exchange over Ethernet

🚀 Quick Start

Basic Usage

#include "sv_encoder.h"
int main() {
// 1. Initialize config
SV_Config config;
sv_config_init(&config, "MU01_SVID", 0x4000);
// 2. Set sample values (currents in A, voltages in V)
float currents[4] = {100.0f, -50.0f, -50.0f, 0.0f};
float voltages[4] = {11000.0f, -5500.0f, -5500.0f, 0.0f};
sv_set_samples_float(&config.asdu[0].samples, currents, voltages);
// 3. Set other parameters
config.asdu[0].smpCnt = 0;
config.asdu[0].smpSynch = SV_SYNC_GLOBAL;
// 4. Build Ethernet frame
uint8_t frame[1518];
uint8_t srcMAC[6] = {0x00, 0x11, 0x22, 0x33, 0x44, 0x55};
size_t len = sv_build_frame(frame, sizeof(frame),
&config, srcMAC, NULL,
4, // Priority
0); // VLAN ID
// 5. Send frame on network (platform-specific)
// send_raw_ethernet(frame, len);
return 0;
}

Continuous Streaming

// For 50Hz, 80 samples per period
const uint16_t SPP = 80;
while (running) {
// Get current measurements
float currents[4], voltages[4];
read_measurements(currents, voltages);
// Update sample data
sv_set_samples_float(&config.asdu[0].samples, currents, voltages);
// Build and send frame
size_t len = sv_build_frame(frame, sizeof(frame), &config, srcMAC, NULL, 4, 0);
send_raw_ethernet(frame, len);
// Increment sample counter (wraps at SPP)
sv_increment_smpCnt(&config.asdu[0], SPP);
// Wait for next sample time (250µs for 50Hz/80SPP)
sleep_us(250);
}

⚠️ Notes

  1. Timing: SV messages must be sent at precise intervals (250µs for 50Hz/80SPP)
  2. Synchronization: Use PTP (IEEE 1588) or IRIG-B for time synchronization
  3. Multicast MAC: Default is 01:0C:CD:04:xx:xx where xx:xx comes from APPID
  4. VLAN: Typically SV uses dedicated VLAN with priority 4-7
